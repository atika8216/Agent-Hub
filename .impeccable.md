## Design Context

### Users
- **Primary**: Internal teams (data engineers, data scientists, business analysts) who
  interact with Databricks AI agents for their daily tasks.
- **Secondary**: Admins who manage which agents are visible and configure memory settings.
- **Context**: Mixed desktop + mobile. Desktop is the heavy-use surface during work hours; the
  mobile layout is a true citizen for in-meeting lookups, not a resized desktop.

### Use Cases
- Browse available AI agents and understand what each does
- Check whether they have access to a specific agent and its sub-components
- Chat with agents in real-time with streaming responses
- Resume previous conversations across sessions
- Admin: manage catalog visibility and memory configuration

### Brand Personality
- **Clear**: Plain-language labels, never jargon where a noun will do.
- **Confident**: Uses color and motion sparingly because the content is the product.
- **Uncluttered**: Only what you need on the current step. The next step is one tap away.

### Aesthetic Direction: "Clarity"
- **Concept**: An iOS-native feel for a data-platform app. Calm surfaces, purposeful color,
  immediate feedback. The catalog -> chat flow should feel like opening iMessage with an
  agent -- continuous, familiar, effortless.
- **Brand voice**: Clear. Confident. Uncluttered.
- **Theme model**: Dual theme. System-follow by default; user can override to Light or Dark.
  Persisted per-user via `user_prefs`. No dark-only cockpit mode.
- **Temperature inversion from "Observatory"**: Neutrals are **warm** (~40deg hue) at low
  chroma instead of cool indigo. Dark mode is a quiet warm-charcoal space, not a deep-space
  console. The existing warm red-orange accent stays for brand-forward moments (primary CTA,
  user bubble) and is now paired with **iOS system blue** for chrome -- focus rings, nav
  selection, links. Two accents with clear roles, not a single hot accent for everything.
- **Anti-references**: Cockpit / instrument-panel aesthetics, dark glowing accents, cool
  indigo-on-black, glassmorphism as decoration, heavy drop-shadows, gradient text, hot pinks
  masquerading as "vibrant".
- **Key differentiator**: The catalog-to-chat transition feels like opening a conversation in
  iMessage -- continuous, calm, familiar.
- **Depth model**: Three surface tiers (canvas / surface / elevated) via 1-3 OKLCH L-steps.
  Light mode adds a single 1px shadow at surface-elevated; dark mode uses an inset 1px top
  hairline instead of a shadow so elevated cards don't glow.

### Typography
- **System-first stack** so Apple devices get SF Pro automatically; other platforms fall back
  to **Pretendard Variable** (open-source, iOS-adjacent) and finally `system-ui`. Monospace
  stays for endpoint names (`SF Mono` on Apple, `JetBrains Mono` elsewhere).
- **Headlines / labels**: same family as body; hierarchy via weight + size + letter-spacing,
  never via a different font family.
- **Type scale (fixed rem, per /impeccable rules)**:
  - display-1: 1.75rem / 700 / -0.025em
  - display-2: 1.375rem / 600 / -0.02em
  - title:     1.0625rem / 600 / -0.01em
  - body:      0.9375rem / 400 / 1.5 line-height
  - subhead:   0.8125rem / 500 / +0.005em
  - caption:   0.75rem / 500 / +0.01em
- **Removed**: Satoshi + Switzer (Fontshare) -- tied to the retired Observatory direction.
- **Banned**: Inter, Manrope, Roboto, Open Sans, Space Grotesk, Work Sans, Syne.

### Color System
- All colors in OKLCH.
- **Neutrals are warm-tinted** (~40deg hue at 0.003-0.007 chroma). No pure grays.
- **Two accents**:
  - `--accent-brand` (warm red-orange, ~30deg hue) -- primary CTAs, the user's own message
    bubble in chat, destructive-confirm states.
  - `--accent-chrome` (iOS system blue, ~250deg hue at moderate chroma) -- focus rings, nav
    selection, links, info toasts.
- **Badge style**: 10% tinted backgrounds with strong text, iOS-style pill radius, never a
  solid saturated background.
- **Accent usage**: Each surface has at most one hot-accent element in view at a time. If
  two would appear, one downgrades to the neutral treatment.

### Motion
- Default easing `cubic-bezier(0.25, 0.1, 0.25, 1)` (iOS default). Emphasize easing gets a
  tiny overshoot: `cubic-bezier(0.2, 0, 0, 1.05)`.
- Durations: 120ms hover, 200ms enter, 260ms page transitions.
- `@media (prefers-reduced-motion: reduce)` zeroes transforms, keeps opacity at 120ms.
- Only animate `transform` + `opacity`.

### Design Principles
1. **Access clarity first** -- a user should know within 1 second if they can use an agent.
2. **Content first, chrome second** -- the catalog card, the chat bubble, the tool-call row
   are the UI; the shell frames them quietly.
3. **Progressive disclosure** -- agent overview in catalog; sub-components and privilege
   details on drill-in; raw JSON on expand.
4. **Continuity** -- catalog browse to chat stream is zero-friction and keeps state across
   the URL transition.
5. **Status at a glance** -- memory mode, connection status, agent type, and access state
   are visible without a tooltip or a dig.
