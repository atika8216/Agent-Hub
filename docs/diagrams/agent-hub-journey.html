App journey

From Agent Discovery to Streamed Answer

The primary product journey moves from a governed agent catalog into a persistent, streaming chat session. Under the surface, the UI updates optimistically, FastAPI persists state in Lakebase, and Databricks APIs perform access checks, agent calls, SQL execution, Genie polling, and MCP/tool orchestration.

Primary userInternal employee searching for the right AI agent and asking follow-up questions.
Admin pathCatalog, feature flags, memory mode, tag config, and endpoint visibility controls.
State anchorLakebase keeps conversations, messages, chart artifacts, suggestions, pins, and prefs.
Stream anchorSSE events keep the chat page responsive while backend calls continue.

User
Open hub

Lands in the app shell with system theme, sidebar or mobile tab bar, and current user identity.

Find an agent

Searches, filters by type, scans descriptions, and checks which agents are available.

Check fit

Opens details to understand sub-agents, owner, access state, and starting point.

Ask question

Types a prompt or uses a pinned starter question in the selected agent chat.

Watch answer

Sees status labels, streaming text, tool progress, errors, retry affordances, and stop control.

Use output

Reads markdown, inspects charts, switches to table, downloads CSV, or taps suggestion chips.

Return later

Resumes a thread from the conversation rail and continues with prior context.

Admin only

Configures discovery, visibility, feature flags, memory, tags, and manual endpoints.

React UI
Hydrate shell

Loads /app/config, /me, user prefs, feature flags, and layout navigation.

Catalog route

/catalog renders agent cards, type filters, search, empty states, and Genie cards.

Agent detail

/catalog/$agentId calls agent detail and access hooks, then links to chat.

useChat()

Adds the user message immediately, starts local streaming state, and posts to the chat endpoint.

SSE parser

Reads data: lines, extracts Genie statuses, appends tokens, and updates URL/sidebar on started.

Render enrichments

Message bubble mounts tool blocks, chart hydrator, EChart cards, and suggestion chips.

Reload path

Conversation route populates the store, lazy-loads charts/suggestions, and keeps the scroll anchor pinned.

Admin routes

/admin/catalog and /admin/settings manage feature and catalog surfaces.

FastAPI
Resolve context

Middleware applies request tracing, timeout, security headers, rate limit, CORS, gzip, and versioning.

List catalog

Router delegates agent, Genie, and catalog config requests to typed service functions.

Access probe

Checks user permissions, owner fallback, sub-agent visibility, and feature availability.

Chat preflight

POST /chat/{endpoint} ensures conversation, persists the user message, emits started.

Dispatch stream

chat_service routes by endpoint prefix/type: serving, Genie, UC HTTP, or MCP.

Persist output

Assistant message, chart artifacts, suggestions, tool events, and conversation title/touch are saved.

Rehydrate APIs

/conversations, /messages/{id}/charts, and /messages/{id}/suggestions rebuild the UI.

Admin APIs

Discovery, rescan, manual UC endpoints, tag config, feature flags, and memory mode endpoints.

Lakebase
Prefs + flags

user_prefs, admin_settings, and user_roles shape the shell and permissions.

Catalog cache

catalog_config stores display names, descriptions, visibility, owners, type, and metadata.

Access state

Owner email and metadata support fallback access and route-specific display behavior.

Conversation row

conversations and messages store the internal thread and first user turn.

Context memory

memory_long_term and recent messages can be assembled into agent context.

Enrichment tables

chart_artifacts, suggestions_cache, and pin event records support durable reloads.

Conversation list

Updated timestamps and previews rebuild the sidebar and recent-conversation index.

Rollout controls

Feature flags and memory settings are data flips, not code redeploys.

Databricks
Apps runtime

Serves the Vite build and FastAPI app with declared OBO scopes, health checks, and env vars.

Discovery APIs

Tiles, serving endpoints, Genie Spaces, UC tags, function tags, and connection metadata.

Permission APIs

OBO WorkspaceClient probes serving endpoints and Genie access, with owner fallback where needed.

Agent call

Model Serving/MAS/KA invocation, Genie conversation API, UC HTTP connection, or MCP tool call.

Execution plane

Genie polls status, SQL Warehouse runs queries, MCP tools return observations, serving streams text.

Data results

Genie query attachments become tables and ECharts options; tool results become timeline events.

Workspace continuity

Databricks identities, endpoints, spaces, and UC names remain stable across app sessions.

Admin workspace

SQL warehouse and service principal reads power catalog discovery and metadata refresh.
